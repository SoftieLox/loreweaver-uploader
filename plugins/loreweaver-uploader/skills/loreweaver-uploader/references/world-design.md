# LoreWeaver world design

Build in dependency order: premise, player role, opening pressure, rules, factions, locations, characters, relationships, items/events, progression, then polish.

Each major character needs a goal, pressure, leverage, limitation, voice, relationships, and reason to engage the player. Each location needs purpose, atmosphere, active pressure, and connected entities. Each faction needs objective, method, resources, internal tension, and external stance.

Write lore cards for retrieval:

- Put the identifying fact early.
- Use concrete names and aliases as keywords.
- Keep one primary subject per card.
- Separate public fact from hidden truth when the live editor supports visibility.
- Avoid duplicated canonical facts across cards; link instead.
- Assign importance by narrative consequence, not by affection for the paragraph.

Openings should establish an immediate situation requiring player input without determining the player's reaction or choice.
