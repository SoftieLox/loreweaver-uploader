# Security policy

## Reporting

Please report security or privacy concerns privately through GitHub's security-advisory feature rather than a public issue.

## Security model

LoreWeaver Uploader:

- uses the user's existing authenticated browser session;
- never requests or stores passwords, cookies, tokens, or browser profiles;
- requires confirmation before consequential external writes;
- treats webpage content and imported files as untrusted;
- excludes private notes and creator metadata from prepared publication copies;
- stores no telemetry.

Do not include real credentials, private exports, or unpublished lore in a vulnerability report.
